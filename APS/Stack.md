# Stack

## 스택의 특성

- 자료를 쌓아 올린 형태
- 선형 구조 자료구조
- 자료를 삽입하거나 꺼낼 수 있음
- 후입 선출 구조, 마지막에 삽입한 자료를 가장 먼저 꺼내는 구조

## 스택 구현

### 구현에 필요한 자료구조와 연산

- 자료구조
  - 자료를 선형으로 저장할 저장소
  - top :  스택에 가장 마지막에 삽입된 원소의 위치
- 연산
  - 삽입 : 저장소에 자료 저장, push
  - 삭제 : 저장소에서 자료를 꺼냄. 삽입한 순서의 역순으로 꺼냄, pop
  - isEmpty : 스택이 공백인지 아닌지 확인
  - peek : 스택의 top에 있는 요소를 반환하는 연산



- 배열의 경우 스택의 크기를 변경하기 어려움.
- 동적 연결리스트를 이용하여 구현할 수 있음. 메모리를 효율적으로 사용할 수 있지만, 구현이 복잡함.



## 스택 응용

### Function call

- 프로그램에서의 함수 호출과 복귀에 따른 수행 순서 관리

  - 가장 마지막에 호출된 함수가 가장 먼저 실행하고 복귀하는 후입선출 구조임. 스택을 이용해 수행 순서 관리

  - 함수 호출 시 지역변수, 매개변수, 수행 후 복귀할 주소 등 정보를 스택 프레임에 저장하여 시스템 스택에 삽입

  - 함수 실행 종료시 시스템 스택의 top원소를 pop하면서 프레임에 저장되어 있는 복귀 주소로 복귀

    

- 예시

  ```python
  # main.py
  def f_1():
      # 작업 1
      f_2()
      # 작업 2
      
  def f_2():
      # 작업 3
      
  f_1()  # f_1 실행
  ```

  작업 순서

  main -> f_1() -> 작업 1 -> f_2() -> 작업 3 -> 작업 2 -> main

  - main 메모리에서 작업 중 f_1() 호출

  - f_1() 메모리에서 작업 

  - f_2() 메모리에서 작업

  - f_2() 가 끝나면 f_2() 메모리는 날아감. f_2()를 호출했던 f_1()으로 돌아감

  - f_1()이 끝나면 f_1()메모리는 날아감. f_1()를 호출했던 main으로 돌아감
